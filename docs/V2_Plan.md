# Stock Pudding v2 — 계획 & 진행

> 벤치마크: [인베스팅닷컴](https://kr.investing.com) 기술 분석 페이지.
> 그들의 **개념**(인터랙티브 차트, 다양한 기술 지표, 신호 요약)을 가져오되,
> 표현은 스톡푸딩답게 — 쉬운 말, 매매 권유 없음, 캐주얼한 톤.

## v1의 한계 (사용자 피드백, 2026-07-07)

1. 진단 문구 템플릿이 사실상 3개 → 어떤 종목이든 비슷한 출력, 금방 지루해짐
2. 엔진이 1년치 OHLCV를 받아놓고 계산 후 버림 — 데이터 활용도 낮음
3. 차트가 없어 "장난감" 인상, 시각적 다양성 부족
4. "60일선 위" 같은 용어를 주린이가 이해 못 함 (타겟과 모순)

## v2 스코프

### M1 — 완료 (2026-07-07, `feat/web-v2`)

| 기능 | 내용 | 상태 |
|------|------|------|
| 인터랙티브 차트 | 1년 종가 + 이평선(20/60/120) + 거래량 + RSI 패널, 토글 칩, 드래그/줌 | ✅ |
| 기술 지표 6종 추가 | MACD, 스토캐스틱, Williams %R, ROC, 볼린저 %B, ATR | ✅ |
| 기술 신호 요약 | 긍정/중립/부정 집계 + 우세 판정 한줄 (매수/매도 아님 — 신호 우세만) | ✅ |
| 시나리오 진단 사전 | 지표 조합 17개 시나리오 + 수치 삽입 — 종목마다 다른 이야기 | ✅ |
| 지표 용어 도움말 | 모든 지표(12종)에 "?" 토글 — 생활 언어 설명 | ✅ |

구현 메모:
- 차트: **lightweight-charts v5** (TradingView OSS, gzip ~56KB). 분석 페이지에서만 lazy 로드해 랜딩 번들 영향 없음. 멀티 패널은 v5 `addSeries(..., paneIndex)` 사용
- 거래량 막대는 한국 관례 색(상승 빨강/하락 파랑) — 신호등 색과 혼용 아님 (DesignSystem §2.4)
- 기술 지표 6종은 **종합점수에 미반영** (보조 섹션). 쉬운 6지표 = 메인 점수, 프로 지표 = 더 보기 — 주린이 우선 원칙 유지
- 시나리오 사전은 `engine/scoring.py _diagnose_scenario` — 구체적 조합부터 매칭, 미매칭 시 최고/최저 지표를 수치와 엮는 폴백
- v1 캐시 리포트에는 series/technicals가 없으므로 프론트 타입은 optional — 캐시 TTL(30분) 지나면 자연 교체

### M2 — 후보 (다음 스프린트, 우선순위 미정)

- 캔들 차트 모드 + 기간 선택 (3달/1년/3년)
- 점수 히스토리: Supabase reports 축적분으로 "지난주 64 → 오늘 59" 변화 표시
- P1 랭킹: 오늘 많이 본 종목 / 도움된 분석 TOP (reports·votes 집계)
- 시장/업종 대비 상대화 ("코스피 평균 PBR 대비 낮음")
- 시나리오 사전 확장 (계절성·연속 상승/하락일 등 컨텍스트 추가)
- 푸딩 게이지 브랜딩 (점수를 푸딩 상태로 — 탱글탱글/흔들흔들/무너짐)

## 원칙 (변하지 않는 것)

- 매매 권유 문구 금지 — "긍정/부정 신호 우세"까지만, 매수/매도 단어 사용 안 함
- 해석 텍스트는 전부 규칙 기반 (LLM 생성 금지, PRD)
- 지표 계산은 Python 엔진, 프론트는 표시만
- 쉬운 6지표가 메인, 전문 지표는 원하는 사람만 펼쳐보는 구조
