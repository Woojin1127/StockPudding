# Stock Pudding — 기능명세서

> 기능 단위로 "무엇을 만들지 / 어디까지 됐는지"만 관리하는 문서.
> 배경·타겟·유저저니는 `StockPudding_PRD.md`, UI 스타일은 `DesignSystem.md`, 날짜별 작업 로그는 `PROGRESS.md` 참고.
> 기능이 추가/변경/완료될 때마다 이 문서만 갱신한다.

`완료` · `진행중` · `대기` · `설계 전`(P2, 아직 상세 정의 안 함)

## P0 상세

### F1. 종목 검색
- 입력: 종목명(한글) 또는 종목코드(숫자) 문자열
- 출력: `{code, name, market}[]`
- 완료 기준: 프론트 검색바 입력 → 엔진 `/search` 호출 → 결과 리스트 표시 → 클릭 시 F3로 이동
- 현재: 엔진 API는 동작(전종목 2875개 캐싱), 프론트는 UI만 있고 실제 호출 안 함(`LandingPage`의 `SearchBar`는 로컬 상태만 다룸)

### F2. 종목 분석 (6개 지표)
- 지표: RSI, 52주 고점 대비 낙폭, 이평선(20/60/120일) 대비 현재가, 거래량(평균 대비), PBR, 부채비율
- 출력: 종합점수(0~100) + 신호등(🟢/🟡/🔴) + 지표별 값/신호/설명 + 한줄 진단
- 완료 기준: `/analyze/{code}` 호출 → 6개 지표 전부 값 존재, 매매 권유 문구 없음, 진단 끝에 "투자 판단은 직접 하세요" 고정 문구
- 현재: 엔진에서 완료, 삼성/카카오/SK하이닉스 3종목 검증됨

### F3. 분석 결과 화면
- 구성: ScoreCard(종합점수+신호등) + IndicatorCard×6 + SignalSummary(신호 요약 2~3개) + 한줄 진단
- 완료 기준: F1에서 종목 선택 시 F2 결과를 DesignSystem.md §5.2~5.5 스펙대로 렌더링, 로딩/에러 상태 포함
- 현재: 미착수. 랜딩 페이지의 "예시 카드"는 정적 더미 데이터일 뿐 실제 연동 아님

### F4. 리포트 저장
- 완료 기준: 분석 결과가 고유 ID로 Supabase에 저장되고, 그 ID로 재접근 가능(공유 링크 대비)
- 현재: 미착수. Supabase 프로젝트/스키마 자체가 아직 없음

### F5. 좋아요/싫어요 투표
- 완료 기준: 투표 클릭 시 즉시 수치 반영(낙관적 업데이트), 같은 기기에서 중복 투표 방지, 재클릭 시 취소
- 현재: 미착수

### F6. 분석 결과 캐싱
- 완료 기준: 같은 종목을 짧은 시간 내 재검색하면 Supabase에 저장된 값을 재사용하고 엔진을 다시 호출하지 않음, TTL 경과 후 자동 갱신
- 현재: 미착수

## P1 상세

- F7 인기 분석 랭킹: 좋아요 순 "도움된 분석 TOP", 싫어요 순 "별로였던 분석 TOP" — 종목명/점수/투표수 노출
- F8 오늘 많이 본 종목: Supabase 집계 기반
- F9 최근 검색: 로그인 없이 로컬(localStorage) 저장

세부 완료 기준은 P0 구현 후 정의.

## P2 (설계 전)

- F10 로그인 & 관심 종목, F11 알림, F12 정확도 검증 — PRD 6장 기준 이름만 확정, 상세 스펙은 P0/P1 완료 후 작성.
